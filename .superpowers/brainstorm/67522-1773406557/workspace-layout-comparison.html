方案 A：Workspace = Window

每个 Workspace 是独立窗口，Sidebar 跨窗口导航。完全保留原生 macOS Tab，对 Ghostty 上游零侵入。切换 Workspace = 切换窗口。

Pros

  • 原生 Tab 零改动
  • Split 机制零改动
  • 上游追踪极简单
  • macOS 原生多窗口习惯

Cons

  • 切换有窗口动画
  • Sidebar 需跨窗口同步

方案 B：Workspace 内嵌于 Window

所有 Workspace 在同一窗口内切换。需要自定义 Tab Bar（放弃原生 macOS Tab），切换无窗口动画，更像 IDE。

Pros

  • 切换更流畅 (<300ms)
  • 单窗口 IDE 体验

Cons

  • 必须自己实现 Tab Bar
  • 后台 PTY 管理复杂
  • 对上游代码改动大
  • BaseTerminalController 需重构

关键区别

方案 A 的 Tab Bar 是 macOS 原生的（在标题栏里），切换 Workspace 切换窗口。
方案 B 的 Tab Bar 是自定义的（标题栏下方），切换 Workspace 在同一窗口内完成。

两者的 Sidebar、File Browser、Terminal 分屏、Status Bar 布局完全相同。
